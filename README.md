# Naboo

Naboo ou **N**ote et **A**nalyse du **B**udget **O**rganisé et **O**ptimisé est une application de gestion de budget personnel. Elle permet aux utilisateurs de suivre leurs dépenses, de planifier leur budget et d'analyser leurs habitudes financières.

## Stack Technique

| Composant | Technologie |
|-----------|------------|
| **Frontend** | React.js avec TypeScript pour une interface utilisateur réactive et moderne (Vite) |
| **Backend** | PHP Laravel pour une API robuste et sécurisée |
| **Base de données** | PostgreSQL pour une gestion efficace des données |
| **Authentification** | Laravel Sanctum pour une authentification sécurisée |
| **Déploiement** | Docker pour une containerisation facile et une portabilité accrue |
| **CI/CD** | GitHub Actions pour automatiser les tests et le déploiement |
| **Interfaces** | Tailwind CSS pour un design moderne et responsive, ShadcnUI pour des composants réutilisables |

## Fonctionnalités

- **Suivi des dépenses** : Les utilisateurs peuvent enregistrer leurs dépenses quotidiennes et les catégoriser pour une meilleure organisation.

- **Planification du budget** : Les utilisateurs peuvent définir des budgets mensuels pour différentes catégories de dépenses et suivre leur progression.

- **Historique des transactions** : Les utilisateurs peuvent consulter l'historique de leurs transactions pour analyser leurs habitudes de dépenses.

- **Rapports et analyses** : L'application génère des rapports visuels pour aider les utilisateurs à comprendre leurs habitudes financières et à prendre des décisions éclairées.

## Points Importants
- **Sécurité** : L'application utilise Laravel Sanctum pour assurer une authentification sécurisée et protéger les données des utilisateurs.
- **Performance** : L'utilisation de Vite pour le frontend garantit une expérience utilisateur fluide et rapide.
- **Scalabilité** : La structure modulaire de l'application permet une évolution facile et l'ajout de nouvelles fonctionnalités à l'avenir.
- **Simplicité** : L'interface utilisateur est conçue pour être intuitive et facile à utiliser, même pour les utilisateurs novices en gestion de budget.
- **Page d'accueil** : La page principal doit être la plus simple possible, avec un accès rapide à la création d'une nouvelle transaction et à la visualisation du budget actuel.
- **Mobile-first** : L'application doit être conçue en priorité pour les appareils mobiles, avec une interface responsive qui s'adapte à différentes tailles d'écran.
